## SEÇÃO 1: REESTRUTURAÇÃO DE TIME

### 1.1 Time de Vendas - Ação Imediata

#### Diagnóstico:
```
ATUAL: 45+ vendedores (setters + closers)
ESTIMADO NECESSÁRIO: 25-30 vendedores
EXCESSO: 15-20 vendedores (~35-45%)
```

#### Plano de Ação:

**SEMANA 1-2: Análise de Performance**
```
Para cada vendedor, calcular:
├── Receita gerada (últimos 90 dias)
├── Número de calls realizadas
├── Taxa de conversão individual
├── Custo de comissão pago
└── ROI = Receita ÷ Comissão

Classificar em:
├── A-Players (Top 20%): >150% da média de conversão
├── B-Players (Middle 60%): 80-150% da média
└── C-Players (Bottom 20%): <80% da média
```

**SEMANA 3-4: Primeira Rodada de Cortes**
```
AÇÃO: Desvincular 100% dos C-Players
ESTIMATIVA: 9-10 vendedores
ECONOMIA IMEDIATA: $13,500-20,000/mês em comissões
RISCO: Baixo (esses vendedores já não performam)
```

**SEMANA 5-8: Redistribuição**
```
AÇÃO: Redistribuir leads dos C-Players entre A e B-Players
RESULTADO ESPERADO: Aumento de conversão geral
MONITORAR: Se A-Players mantêm performance com mais volume
```

**SEMANA 9-12: Segunda Rodada (com Automação)**
```
AÇÃO: Implementar Setter Virtual
SUBSTITUIR: 5-10 setters humanos adicionais
MANTER: Apenas closers A e B-Players
ESTRUTURA FINAL: 15-20 vendedores humanos + IA
```

#### Estrutura Final Proposta:

```
ANTES:                          DEPOIS:
45+ vendedores                  20 vendedores
├── ~23 setters                 ├── Setter Virtual (IA)
├── ~22 closers                 ├── 5 setters humanos (backup/complex)
└── Custo: $45-90k/mês          ├── 15 closers A/B-players
                                └── Custo: $25-35k/mês

ECONOMIA: $20-55k/mês
```

### 1.2 Estrutura de Incentivos do Benny

#### Problema Identificado:
```
ATUAL: Benny ganha comissão sobre o time de vendas
INCENTIVO: Mais vendedores = mais comissão base
RESULTADO: Time inchado, baixa eficiência
```

#### Recomendação:

**Opção A: Mudar para Comissão sobre Resultado**
```
NOVO MODELO:
├── Base fixa: $3,000/mês
├── Comissão: 2-3% da receita FECHADA pelo time
└── Bônus: $500 por cada vendedor que atinge meta individual

INCENTIVO NOVO: Time menor e melhor = mais receita = mais comissão
```

**Opção B: Tornar Benny Sócio Operacional**
```
NOVO MODELO:
├── Salário fixo: $5,000/mês
├── Equity: 2-5% do lucro da operação de vendas
└── Vesting: 2 anos

INCENTIVO NOVO: Pensar como dono, otimizar margem
```

**Recomendação do Board:** Opção A para curto prazo, migrar para Opção B se Benny performar.

└── RESULTADO: Adam produz 3-5x mais sem contratar

PRIORIDADE 3: Otimizar Lucas
├── Cortes automáticos → Tools de IA
├── Legendas → Transcrição automática
├── Redimensionamento → Scripts automatizados
└── RESULTADO: Lucas foca no criativo, não no repetitivo

## SEÇÃO 2: MUDANÇAS NO MODELO DE COMISSIONAMENTO

### 2.1 Análise do Modelo Atual

```
MODELO: 20% do (Faturamento - Ads)

PRÓS:
✓ Simples de calcular
✓ Alinha com resultado do cliente
✓ Sem custo fixo para o cliente

CONTRAS:
✗ Margem cai quando cliente escala com ads
✗ Incentiva agência a evitar estratégias de escala com ads
✗ Receita imprevisível
✗ Risco de margem zero em clientes com alto ad spend
```

### 2.2 Recomendação: Manter com Critérios

**Curto Prazo (0-6 meses):** NÃO mudar o modelo

Justificativa:
> "Mudar comissionamento agora vai criar atrito desnecessário com clientes existentes. Vocês não têm leverage suficiente. Primeiro, otimizem internamente."

**Implementar IMEDIATAMENTE:** Critérios de Aceitação de Novos Clientes

```
NOVO CLIENTE SÓ ENTRA SE:

1. Ad Spend Projetado < 40% do faturamento
   └── Protege margem mínima de 12% efetivo

2. Ticket Médio > $1,500
   └── Garante unit economics saudável

3. Audiência Existente > 10,000 seguidores OU lista > 5,000
   └── Reduz dependência de ads frios

4. Expert com Prova Social Verificável
   └── Aumenta conversão, reduz custo de aquisição

5. Comprometimento Mínimo de 6 meses
   └── Amortiza custo de onboarding
```

**Médio Prazo (6-12 meses):** Migrar para Híbrido

```
NOVO MODELO PARA CLIENTES NOVOS:

Base Fixa: $10,000-20,000/mês (dependendo do tamanho)
Performance: 10% do (Faturamento - Ads)

BENEFÍCIOS:
├── Receita garantida mesmo em meses ruins
├── Cliente ainda tem skin in the game
├── Protege margem da agência
└── Alinha crescimento de ambos
```

## SEÇÃO 3: PRIORIZAÇÃO DE CLIENTES

### 3.1 Framework de Classificação

Sem dados específicos de cada cliente, o Board recomenda aplicar este framework:

```
MATRIZ DE PRIORIZAÇÃO:

              │ ALTO POTENCIAL │ BAIXO POTENCIAL │
──────────────┼────────────────┼─────────────────┤
ALTA MARGEM   │ ★★★ INVESTIR   │ ★★ MANTER       │
──────────────┼────────────────┼─────────────────┤
BAIXA MARGEM  │ ★ RENEGOCIAR   │ ✗ DESCONTINUAR  │
──────────────┴────────────────┴─────────────────┘

DEFINIÇÕES:
- Alto Potencial: Pode fazer $500k+/mês
- Baixo Potencial: Teto de $200-300k/mês
- Alta Margem: Ad spend < 40% do faturamento
- Baixa Margem: Ad spend > 40% do faturamento
```

### 3.2 Ações por Categoria

**★★★ INVESTIR (Provavelmente 1-2 clientes)**
```
AÇÕES:
├── Alocar melhores vendedores
├── Priorizar tempo de Rodrigo e Adam
├── Aumentar budget de criação de conteúdo
├── Implementar automações primeiro
└── META: Maximizar receita desses clientes
```

**★★ MANTER (Provavelmente 2 clientes)**
```
AÇÕES:
├── Manter operação normal
├── Não aumentar investimento
├── Automatizar ao máximo para reduzir custo
└── META: Lucro estável com mínimo esforço
```

**★ RENEGOCIAR (Provavelmente 1-2 clientes)**
```
AÇÕES:
├── Apresentar novo modelo híbrido
├── Exigir mínimo garantido mensal
├── Se recusarem: preparar transição de saída
└── META: Transformar em margem positiva ou sair
```

**✗ DESCONTINUAR (0-1 cliente)**
```
AÇÕES:
├── Comunicar com 30 dias de antecedência
├── Oferecer transição suave
├── Não renovar contrato
└── META: Liberar recursos para clientes melhores
```
