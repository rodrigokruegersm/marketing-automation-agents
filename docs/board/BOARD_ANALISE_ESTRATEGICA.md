## SESSÃO 1: DIAGNÓSTICO BRUTAL DA OPERAÇÃO

### ALEX HORMOZI abre a discussão:

> "Vocês têm um problema estrutural grave. O modelo de 20% pós-ads é uma armadilha. Se o cliente gasta 90% em ads, vocês ficam com migalhas. Isso cria incentivos desalinhados - vocês querem que o cliente gaste MENOS em ads, mas isso pode significar MENOS escala pra ele."

**Diagnóstico financeiro:**
- Cliente faz $1M com $0 ads → Agência ganha $200k (20%)
- Cliente faz $1M com $900k ads → Agência ganha $20k (2% efetivo)
- **PROBLEMA:** Margem da agência é INVERSAMENTE proporcional à escala do cliente

### SAM OVENS responde:

> "O modelo precisa ser híbrido. Base fixa + performance. Caso contrário, vocês estão subsidiando o crescimento do cliente com a própria margem."

**Proposta de reestruturação:**
```
MODELO ATUAL:        20% × (Faturamento - Ads)
MODELO PROPOSTO:     $15k base + 10% × (Faturamento - Ads)
```

**Simulação comparativa:**

| Cenário | Faturamento | Ads | MODELO ATUAL | MODELO PROPOSTO |
|---------|-------------|-----|--------------|-----------------|
| A | $500k | $50k | $90k | $60k |
| B | $500k | $200k | $60k | $45k |
| C | $1M | $100k | $180k | $105k |
| D | $1M | $500k | $100k | $65k |
| E | $1M | $800k | $40k | $35k |

> "Vejam: no modelo proposto vocês ganham MENOS nos cenários bons, MAS ganham previsibilidade e proteção nos cenários ruins. Com $15k garantido × 6 clientes = $90k de receita FIXA antes de qualquer performance."

### COLE GORDON entra na discussão de vendas:

> "45 setters e closers para 6 clientes? Isso é uma média de 7.5 vendedores por cliente. Qual é o ticket médio? Qual a taxa de conversão? Quantos leads por dia cada setter processa?"

**Perguntas críticas que o board precisa responder:**

1. **Qual o CAC (Custo de Aquisição de Cliente) por projeto?**
2. **Qual o LTV médio dos compradores dos clientes?**
3. **Quantos calls de vendas acontecem por dia/semana?**
4. **Qual a taxa de show-up rate?**
5. **Qual a taxa de conversão setter→closer→venda?**

> "Se vocês não têm esses números NA PONTA DA LÍNGUA, vocês estão operando no escuro. E se estão no escuro, os agentes de IA não vão resolver - vão amplificar o caos."

### TAKI MOORE sobre operações:

> "O gargalo de vocês não é falta de pessoas. É falta de SISTEMAS. Vocês têm 6 clientes com potencial de $500k-$2M cada. Isso significa que vocês deveriam estar faturando entre $3M e $12M por mês nos clientes. Se a meta é $500k/mês para a agência, vocês precisam de apenas $2.5M em faturamento dos clientes (assumindo modelo atual com média de 20% efetivo)."

**Análise de capacidade:**

| Métrica | Situação Ideal | Pergunta |
|---------|---------------|----------|
| Faturamento necessário clientes | $2.5M-$6.25M/mês | Quanto os clientes faturam HOJE? |
| Número de clientes | 6 | Suficiente se performarem |
| Vendedores | 45 | Estão 100% ocupados ou ociosos? |
| Custo fixo equipe | ~$12.5k/mês | Está otimizado? |

### RUSSELL BRUNSON sobre o mercado:

> "Mercado 'How to make money' está saturado, mas ainda é o maior. A questão não é SE dá dinheiro, é COMO vocês se posicionam. Coprodução funciona quando o especialista tem AUTORIDADE REAL. Vocês estão escolhendo os especialistas certos?"

**Framework de seleção de clientes:**

```
SCORE DE CLIENTE IDEAL (1-10):

[ ] Autoridade comprovada (seguidores, resultados, prova social)
[ ] Lista existente (email, WhatsApp, Telegram)
[ ] Produto validado (já vendeu antes)
[ ] Margem saudável (ticket > $997 ou recorrência)
[ ] Disponibilidade (tempo para criar conteúdo, aparecer)
[ ] Alinhamento (visão de longo prazo, não só dinheiro rápido)
```

## SESSÃO 5: ESPECIFICAÇÃO DOS AGENTES

### AGENTE 1: RADAR (Captação e Monitoramento)

**Função:** Monitorar fontes de leads, identificar oportunidades, alertar equipe

**MCPs necessários:**
- `mcp-server-slack` - Integração com Slack
- `mcp-server-fetch` - Web scraping e monitoramento
- Custom MCP para APIs de ads (Meta, Google)

**Inputs:**
- Dados de campanhas de ads
- Formulários de landing pages
- Mensagens em grupos/comunidades

**Outputs:**
- Lead qualificado com score
- Alerta para setter designado
- Update automático no CRM

**Métricas:**
- Tempo de processamento: <1 minuto
- Acurácia de qualificação: >85%
- Leads processados/dia: ilimitado

---

### AGENTE 2: SETTER VIRTUAL (Qualificação)

**Função:** Primeiro contato, qualificação BANT, agendamento

**MCPs necessários:**
- `mcp-server-calendar` - Google Calendar/Calendly
- `mcp-server-whatsapp` - Integração WhatsApp Business
- `mcp-server-twilio` - SMS e chamadas

**Fluxo:**
```
Lead entra → Contato em <5min → Qualificação BANT →
Score ≥7 → Agenda call com closer →
Score <7 → Nurturing automatizado
```

**Métricas:**
- First response time: <5 minutos
- Qualificação completa: <15 minutos
- Taxa de agendamento: >40%

---

### AGENTE 3: CLOSER SUPPORT (Suporte a Vendas)

**Função:** Preparar closers para calls, fornecer intel em tempo real

**MCPs necessários:**
- `mcp-server-notion` - Base de conhecimento
- `mcp-server-memory` - Contexto do lead
- Custom MCP para CRM

**Outputs antes da call:**
- Briefing do lead (dores, objeções prováveis, budget)
- Histórico de interações
- Script personalizado baseado no perfil

**Outputs durante a call:**
- Sugestões de rebate de objeções (se integrado)
- Dados em tempo real

**Outputs depois da call:**
- Resumo automático da call
- Next steps definidos
- Update no CRM

---

### AGENTE 4: COPY MASTER (Criação de Conteúdo)

**Função:** Criar e otimizar copies para ads, emails, VSLs

**MCPs necessários:**
- `mcp-server-filesystem` - Acesso a arquivos
- `mcp-server-github` - Versionamento de copies
- Custom MCP para biblioteca de swipes

**Capacidades:**
- Gerar variações de headlines (A/B testing)
- Criar emails de nurturing
- Escrever scripts de VSL
- Adaptar copy para diferentes plataformas

**Métricas:**
- Tempo de criação de campanha: <2 horas (vs. 4-8h manual)
- Variações por campanha: 10+ automaticamente
- CTR médio das copies: tracking contínuo

---

### AGENTE 5: CRM GUARDIAN (Gestão de Dados)

**Função:** Manter CRM atualizado, limpo e acionável

**MCPs necessários:**
- `mcp-server-postgres` ou `mcp-server-sqlite` - Database
- Custom MCP para CRM específico (HubSpot, Pipedrive, Close)
- `mcp-server-slack` - Alertas

**Automações:**
- Update automático após cada interação
- Limpeza de dados duplicados
- Alertas de leads "esquecidos"
- Movimentação automática de pipeline

**Métricas:**
- Dados desatualizados: 0%
- Leads sem ação >48h: alerta automático
- Duplicados: remoção automática

---

### AGENTE 6: ANALYTICS COMMANDER (Relatórios)

**Função:** Consolidar dados, gerar insights, criar relatórios

**MCPs necessários:**
- `mcp-server-google-sheets` - Planilhas
- `mcp-server-puppeteer` - Screenshots de dashboards
- Custom MCP para APIs de analytics

**Outputs:**
- Daily report automático (7am)
- Weekly deep-dive (segunda-feira)
- Alertas de anomalias em tempo real
- Dashboards interativos

**Métricas trackeadas:**
- Por cliente: faturamento, CAC, ROAS, conversões
- Por vendedor: calls, conversões, ticket médio
- Por campanha: CPL, CTR, CPA

---

### AGENTE 7: COMMAND CENTER (Orquestrador)

**Função:** Coordenar todos os agentes, escalar para humanos quando necessário

**MCPs necessários:**
- Todos os anteriores
- `mcp-server-sequential-thinking` - Decisões complexas
- Custom MCP para regras de negócio

**Responsabilidades:**
- Rotear tarefas para agentes corretos
- Identificar situações que precisam de humano
- Consolidar outputs de múltiplos agentes
- Manter contexto entre interações
